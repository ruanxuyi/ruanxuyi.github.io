[LeetCode] LC98 - Validate Binary Search Tree

Posted on 02-04-2019   |     |   visitors

Description

Given a binary tree, determine if it is a valid binary search tree (BST).

Assume a BST is defined as follows:

  • The left subtree of a node contains only nodes with keys less than the node’s key.
  • The right subtree of a node contains only nodes with keys greater than the node’s key.
  • Both the left and right subtrees must also be binary search trees.

Thoughts

DFS recursive (bottom-up)

First start with DFS approach since it is easier to write.

The defualt method isValidBST(TreeNode root) has only one argument. We need a helper method to add low and high bound to check if node.val is in range.

The dfs recursive first goes to the far left node of the tree. (pre-order traversal). Since if (root.val <= min || root.val >= max) return false is after the node.left and node.right recursive call. Hence, it is a bottom-up approach.

DFS recursive (top-down)

The top-down approach is very similiar to bottom-up by moving one line of code. It check root.val if in range before any recursive call. This helps for case where node.val violates range check and return false directly without going too deep.

DFS Iterative (in-order traversal)

The DFS iterative approach takes advantage of the special Binary Search Tree Property. The inorder traversal of a valid BST is an ordered list. Hence, we just need to check if the list is non-decreasing by checking if pre.val <= node.val.

DFS Iterative (top-down + preorder traversal)

The top down approach uses preorder traversal. However, this approach requires customized class object for [low, high] range for each node. Could also maintain two stack for [low, high] for each node as well. Code is not implemnted, please take a look at BFS solution below.
