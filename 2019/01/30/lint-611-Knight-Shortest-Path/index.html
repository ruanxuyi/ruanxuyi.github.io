[LintCode] LC611 - Knight Shortest Path

Posted on 01-30-2019   |     |   visitors

Description

Given a knight in a chessboard (a binary matrix with 0 as empty and 1 as barrier) with a source position, find the shortest path to a destination position, return the length of the route.
Return -1 if destination cannot be reached.

Notice
source and destination must be empty.
Knight can not enter the barrier.

Clarification
If the knight is at (x, y), he can get to the following positions in one step:

Example

source = [2, 0] destination = [2, 2] return 2

source = [2, 0] destination = [2, 2] return 6

source = [2, 0] destination = [2, 2] return -1

> Solve problem here

Thoughts

Understanding:

  • a knight can move 8 directions
  • a visited list should be maintained (Not necessary -> can use existing board, visited location set to true)
    • if cannot modify board, use visited set
    • override equals() and hashCode()
      • overriding hashCode() allow $O(1)$ access for Set operation

Corner cases:

  • null or 0 length for board row/col
  • out of boundry
  • very large board (can limit board size)

Since looking for shortesting path, lets use a queue to maintain all knight‚Äôs next possible valid move (not out of bound, is an empty space..(dont forget this)) locations and process it in BFS maner. Return true upon success find matched coord with Destination. All visited cell will be added to visited set to prevent inf loop. Return false is queue is empty.

DFS also work, but might cuse stack overflow when size of board is large.
