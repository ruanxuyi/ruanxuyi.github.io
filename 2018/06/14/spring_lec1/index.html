REST/RESTful API

Representational state transfer (REST) means:

  • Representational: representation of resources, image, text, etc
  • state transfer: resource only transfer state from server to clinet, but the resource remain constant during the transfer

src: https://stackoverflow.com/questions/10418105/what-does-representational-state-mean-in-rest

  • resource: abstraction of data (images/documents, etc)
  • resources can be grouped into collections
  • understand collection -> resource -> sub-collection and sub-resource (slide 22)

Safe/Idempotent API

  • A safe HTTP method will not modify (update/delete) a resource (read-only)
  • An idempotent HTTP method is a HTTP method that can be called many times without different outcomes

HTTP POST is not idempotent since the request will be executed multiple times if call more than once (reserce created with different IDs)

Note: normal operation by PUT is to update exisiting resource. However, when resource (id) not exist, PUT could also be used to create a new resource. But, once created, the resource stay the same with the same PUT request. Hence, it is idempotent but not safe.

https://stackoverflow.com/questions/45016234/what-is-idempotency-in-http-methods

PATCH update some details of an existing resource,
PUT inserts or replaces the old activation entirely.

@RestController, @RequestMapping

@RequestMapping format

  • @RequestMapping ensures that HTTP requests to /greeting are mapped to the greeting() method.
  • @RequestParam binds request parameter name (string) into parameter (variable name) in greeting() method
  • Greeting object is automatically converted to JSON format by Jackson2, which is definied on the class path

Hence,
localhost:8080/greeting returns {"id":2, "content": "Hello, World!"}
localhost:8080/greeting?name=winnie returns {"id":3, "content": "Hello, winnie!"}

WAR (Web application ARchive)

  • use to distribute java-base web app
  • a single compressed file that contains all components required to run the app
  • deploy to external application server

executable JAR file

  • Tomcat application server is embedded within the JAR file (no need of external application server)

Jar vs War

JAR

  • server applicaiton included
  • standalone application w/ embedded server, dependent services (DI)
  • light weight, can be ran on personal computer
  • easier to deploy java -jar *.jar

WAR

  • server applicaiton NOT included (NOT standalone, since it requires external server to run the WAR app)
  • deploy to pre-exisiting Java EE Application server
  • run multiple applications in ONE server

<parent> tag : Project Inheritance

  • without <parent> tag, POM file extends Super POM
  • use <parent> tag to explicitly specify inheritance of parent POM (ex. spring-boot) other than Super POM
  • remove module’s groupid, and version if a module would like to inheritance them from parent’s

<modules> tag : Project Aggregation

  • similiar to Project Inheritance
  • parent declare the inheritance (instead of child cliam parent in tag)

<properties> tag : Project Interpolation and Variables

  • variable uses more than once should be considered to assign as a variable
  • use the <properties> tag
  • example property ${project.groupId}, ${project.version}, ${project.build.sourceDirectory}
