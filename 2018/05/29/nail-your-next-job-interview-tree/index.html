Tree Reconstruction

297. Serialize and Deserialize Binary Tree

https://leetcode.com/problems/serialize-and-deserialize-binary-tree/description/

BFS w/ queue

  • key: for any none null node N: add X if N.left == null or N.right == null

DFS (ONE traversal w/ X or # to mark boundry)

  • any one kind of traversal
  • require more space to store (X or #)

DFS (TWO traversals wo/ # or X)

  • basic binary tree requires TWO traversals to reconstruct

449. Serialize and Deserialize BST

  • as compact as possible (wo/ X or #)
  • difference than #297, #297 require both X and one single traversal result
  • howerver, #449 only need one single traversal result wo/ X (inorder result given implicitly)

Difference between #297 and #449

  • https://leetcode.com/problems/serialize-and-deserialize-bst/discuss/93177/what's-the-difference-between-this-and-297
  • #449 requires serialized result to be as compact as possible (hence, no ‘#’ or ‘X’ was added)
  • #297 binary tree requires at least TWO traversals from (pre/in/post) to reconstruct tree
  • while #449 only requires ONE traversal from (pre/post) to reconstruct, since (in-order) of BST is sorted string, which is implictly given

105. Construct Binary Tree from Preorder and Inorder Traversal

https://leetcode.com/problems/construct-binary-tree-from-preorder-and-inorder-traversal/description/

Observations:

    1. preorder: first element of preorder is always the root of current subtree
    1. inorder: once root found, in inorder array, left of root is left subtree, and right of root is right subtree.
    1. preorder: Once we know the number of elements in left subtree and right subtree, we could find the corresponding roots for left subtree and right subtree from preorder array.
    1. recursively repeat step 1-3.

Implementations:

  • the hardest part of this question is to determine what indices are needed for the helper function
  • inStart and inEnd indicates the range of current subtree in inorder array
  • pStart tells the location of the root for current subtree (preorder[pStart])

ref: https://leetcode.com/problems/construct-binary-tree-from-preorder-and-inorder-traversal/discuss/34538/My-Accepted-Java-Solution
