Top 5 Frequently Used Algorithms: #1 Divide and Conquer

Posted on 02-24-2018   |     |   visitors

basic concept

Given a complex problem, we can break it down into two or more subproblems of same kind, we can further divide the subproblem into even smaller smaller onesâ€¦ The subproblem eventully is small enough to have a direct result/solution. Finally we combine all results of simple form from subproblems to get the solution for original complex problem.

key insights:

  • for sort problem, when n = 1, no sort is required. when n = 2, one comparison done. when n = 3, we need three comparisions. However, when n become very large, it becomes a hard problem at first glance.
  • subproblem is smaller size of big problem, but problem format remain the same
  • we apply divide and conqure technique, until problem size get small enough that we can directly achieve the solution
  • since subproblem always have the same format, recursion becoome the most intitutive way for divide and conqure technique.
  • hence, recurrsion and dvide and conqure is like brother and sisiter

when to use?

A problem which can be solved by divide and conquer usually has following four properties:

  1. subproblem can be easily solved when size is small
  2. the problem can be easily divided into subproblem of same kind with smaller size
  3. solution to subproblems can be easily combined into solution for the complex one
  4. subproblem is independent to each other (no overlap between subproblem)

The third property is the key. If a complex problem fulfill property 1 and 2 but NOT 3, we should consider greedy alogrithm or dynamic programming instead.

The forth property is usally not strictly enforced. Overlap in subproblem could be solved by divide conquer, however, using dynamic programming is preferred when overlap exists in subproblems.

Traverse vs Divide Conquer

  • Result in parameter vs Result in return value
  • Top down vs Bottom up
  • Merge Sort / Quick Sort
  • 90% Binary Tree Problems!
