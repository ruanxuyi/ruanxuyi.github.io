Leetcode35 - Search Insert Position

Posted on 02-12-2018   |     |   visitors

description

https://leetcode.com/problems/search-insert-position/description/

Given a sorted array and a target value, return the index if the target is found. If not, return the index where it would be if it were inserted in order.

You may assume no duplicates in the array.

Assumptions:

  • sorted
  • no duplicates

Follow up:

  • sorted: if not sorted, this question has no meaning
  • duplicates: with duplicates, if ask for first occurance, merge target == nums[mid] into target <= nums[mid] should work. similiarly if ask for last occurance.

Corner case:

  • target insert before nums[0] OR after nums[nums.length - 1]
  • one element in nums

Pits

Pit1: low = mid + 1

since question not only ask for index of target, if found. but it also ask for where to insert if target not exist.

Hence, target != nums[mid] is not sufficient to exclude index mid.

Solutions

Solution 1: scan array from left to right. if nums[i] < target, increment i. if nums[i] >= target, return i.

time: $O(N)$, space $O(1)$

Solution 2: since the array is sorted, use binary search to locate the target. if target not exist, return the right pointer. we also need to handle corner case: if target is smaller than nums[0], we return 0. if target is larger than nums[nums.length - 1], we return nums.length

time: $O(NlogN)$, space $O(1)$

Solution3: simplify version of solution2.

time: $O(NlogN)$, space $O(1)$
