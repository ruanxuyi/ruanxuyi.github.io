Subsequence of string

given string str=[10, 9, 2, 5], find longest increasing subsequence.

What is the definition of subsequence?

According to Wikipedia, a subsequence is a sequence that can be derived from another sequence by deleting some or no
elements without changing the order of the remaining elements.

https://en.wikipedia.org/wiki/Subsequence

Before we get into optimized solution, we first need to understand how the brute force algorithm work, and what the time and space complexity is.

The brute force algorithm will be following:

  • generate all possible subsequences of the given string (time complexity: $O(2^n)$)
  • for each of the subsequence, we check if it is an increasing subsequence (time complexity: $O(n)$)
  • we finally return maxCount

what is the mathmatically formula for this? If we look at it more carefully, for each of the element in the array, there are two choices:

  1. include that number in the subsequence, or
  2. NOT inculde it.

Hence, the count for total possible subsequences became $2^n$.

Step 2 is intutively, we simply need to check for each elements in subsequence to see if it is increasing, which takes $O(n)$ in the worst case.

In conclusion, the total time complexity of brute force approach for LIS problem is $O(n 2^n)$, which is exponentially expensive. Hence, we should consider dynmaically programming approach, which usually good at reducing exponential ($O(2^n)$) time complexity to *polynomial ($O(n^2)$) one.

Dynmaically programming is good, it reduces time complexity from $O(2^n)$ to $O(n^2)$, however, there is an even better approach to reduce time complexity to $O(nlogn)$ with binary search. Details could be referred to link below.
